OPCIÓN 1: Android Studio (Recomendado) - 2 CLICS
────────────────────────────────────────────────

1. Abre Device Manager (derecha en Android Studio)
2. Presiona ▶ para iniciar un emulador
3. En Android Studio: Run > Run 'app'
4. Selecciona el emulador
5. ¡La app se abre automáticamente!

Tiempo total: 5 minutos


OPCIÓN 2: Terminal (PowerShell)
───────────────────────────────

1. Abre PowerShell en C:\Users\olmed\Downloads\CUICATL

2. Verifica el emulador:
   adb devices

3. Instala el APK:
   adb install app/build/outputs/apk/debug/app-debug.apk

4. Abre la app:
   adb shell am start -n com.example.cuicatl/.MainActivity

5. ¡Listo!

                        🆘 SI HAY PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: "Emulador no inicia"
→ Abre Device Manager, presiona ▶, espera 5 minutos

PROBLEMA: "App no se instala"
→ ./gradlew.bat clean
→ Build > Rebuild Project en Android Studio

PROBLEMA: "Pantalla en blanco"
→ Espera 5 segundos, toca Permitir en el diálogo de permisos

PROBLEMA: "App muy lenta"
→ Device Manager > editar emulador > aumenta RAM a 4GB

PROBLEMA: "No veo canciones"
→ Es normal, el emulador no tiene música. La app funciona igual.

PROBLEMA: "Error al compilar"
→ ./gradlew.bat clean
→ File > Sync Now
→ Build > Rebuild Project

Ver detalles completos en: SOLUCION_DE_PROBLEMAS.txt
