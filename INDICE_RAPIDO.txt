                        💡 CONSEJOS ÚTILES
═══════════════════════════════════════════════════════════════════════════════

💡 El mensaje "BUILD SUCCESSFUL" NO ES UN ERROR
    Es lo que quieres ver. Significa todo está bien.

💡 "UP-TO-DATE" en las tareas es BUENO
    Significa Gradle es eficiente y no recompila innecesariamente.

💡 Si quieres cambios, edita y recompila:
    Build > Rebuild Project

💡 Si la app no cambia después de editar:
    ./gradlew.bat clean
    Luego Build > Rebuild Project

💡 Si el emulador va lento:
    Device Manager > editar > aumenta RAM a 4GB

💡 Para ver logs de errores:
    View > Tool Windows > Logcat
    Busca: "com.example.cuicatl"

💡 Para limpiar todo:
    ./gradlew.bat clean
    Cierra Android Studio
    Abre nuevamente
    Build > Rebuild Project
